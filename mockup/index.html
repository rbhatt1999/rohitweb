// rohitweb / redesign / preview · v3 (real three.js + scroll-driven camera)

Code-corridor 3D — camera flies forward as you scroll

Real WebGL via Three.js. Each device frame runs an independent 3D scene: a long corridor of floating wireframe code-blocks, drifting glyph-particles, fog-graded depth, and a hero ▸ arrow extruded in 3D. The camera's z-position is locked to your scroll — scroll, and you literally fly through the corridor. Pattern lifted from how Sébastien Lempens, Bilal Elmossaoui, and Samsy build their portfolios in 2026.
▸ scroll inside either device — the world moves past you. Mouse-move adds parallax on desktop.
refs: sébastien lempens · bilal elmossaoui · samsy · jordan breton · aimee's papercraft — see notes at bottom.

// v3 — real three.js WebGL code-corridor scene; camera dolly tied to scroll position, mouse parallax, fog-graded depth, drifting glyph particles, extruded ▸ hero glyph, and per-card meshes (icosahedron / house) all rendered in WebGL. Inspired by how 2026 dev portfolios use scroll-driven camera paths (Sébastien Lempens, Bilal Elmossaoui, Samsy, Jordan Breton's flying island). Production build will use React Three Fiber + drei + postprocessing (UnrealBloom) + GSAP ScrollTrigger for choreographed scroll sequences and Lenis for smooth inertial scroll. Email: rohit.bhatt.dev@gmail.com. Featured project: sadhak_ai.
